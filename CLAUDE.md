## Communication
- Be concise. Skip preamble and summaries unless asked
- No emojis
- When unsure, ask before acting

## Code
- No dead code — delete it, don't comment it out
- Use strong types and interfaces where possible
- Avoid `any` and `unknown` in TypeScript; prefer precise types or generics
- Use DTO objects for API inputs/outputs, not database models

## Git
- Never force push to main or master
- Never skip hooks (--no-verify)

## Commit message conventions
- Follow conventional commits: `<type>(<scope>): <subject>`
- Types: `feat`, `fix`, `refactor`, `docs`, `style`, `perf`, `test`, `chore`, `ci`, `build`, `revert`
- Subject line: 50–72 characters, imperative mood ("add feature" not "added feature")
- Include a body if the change is complex or non-obvious, wrapped at 72 characters
- Never add "Generated by Claude" or similar attribution in the message
- Use `/create-commit-message` to generate a suggestion before committing
